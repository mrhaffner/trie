Implementing Search Suggestions with a Trie

Introduction

Selecting an efficient method to find search suggestions presents a unique challenge. This challenge is best overcome with a Trie a.k.a. Prefix tree. We can see a simple implementation with the following search bar:

This implementation is efficient. Suggestions are found in O(H*C) time. "H" represents the height of the Trie. "C" represents the character set the Trie supports. Since "C" is limited, it is therefore a constant. The time becomes O(H). The height of the Trie is the same as the length of its longest suggestion. It is not neccesary to support strings of a potentially unlimited length. Most searches are short, so we can set a reasonable max height for our Trie. "H" is now a constant so the find operation runs in constant time. Deletions and inserts are done in O(H) time which reduces to O(1).

The Trie will find all suggestions for our given prefix. This presents our first problem - we do not want to show a large number of suggestions.
